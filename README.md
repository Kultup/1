## 1. Як це працює

Перший напрям -- Syrve → CRM -- у нас вже працює: коли офіціант відкриває стіл, дані про замовлення автоматично потрапляють у картку Deal.

Зараз потрібно налаштувати зворотній напрям:

```
Менеджер змінює Курс або Гостя у сабформі Deal
          |
          v
Zoho CRM надсилає запит на наш сервер
          |
          v
Наш сервер знаходить потрібну касу і передає команду
          |
          v
Плагін на касі оновлює позиції замовлення у Syrve
```

## 2. Що потрібно зробити у Zoho CRM

### 2.1 Додати приховане поле `Syrve Item ID` у сабформи

Коли Syrve надсилає нам дані про замовлення, разом з кожною позицією приходить `itemId` -- унікальний ідентифікатор рядка замовлення. Без нього ми не можемо точно сказати "змінити саме цю позицію" -- наприклад, якщо в замовленні три однакові піци для різних гостей.

Потрібно: у кожній сабформі (Замовлені послуги, Допродаж офіціанта) додати приховане поле `Syrve Item ID` і заповнювати його значенням `itemId` при отриманні вхідного webhook від Syrve.

### 2.2 Налаштувати тригер

Рекомендую **кнопку "Синхронізувати з Syrve"** на картці Deal. Менеджер змінює курси/гостей у сабформі, натискає кнопку -- і зміни йдуть на касу.

Альтернатива -- автоматично при збереженні Deal. Але тоді кожне збереження буде надсилати запит на касу, навіть якщо курс/гість не змінювались.

### 2.3 Надіслати запит на наш сервер

При спрацюванні тригера -- HTTP POST-запит згідно зі специфікацією нижче.

## 3. Наш API -- куди і що надсилати

### Endpoint

```
POST https://middleware.krainamriy.fun/api/update-order-items
```

### Заголовки

| Заголовок | Значення |
|-----------|----------|
| `Content-Type` | `application/json` |
| `X-API-Key` | `2f72839a224f4544963c9eaa7abcc2f6` |

### Тіло запиту

| Поле | Тип | Опис | Де взяти в CRM |
|------|-----|------|----------------|
| `orderId` | `string (UUID)` | ID замовлення у Syrve | Поле Deal зі Syrve Order ID |
| `city` | `string` | Місто закладу | Поле "Місто" або "Заклад" у Deal |
| `items` | `array` | Список позицій | Рядки сабформи |
| `items[].itemId` | `string (UUID)` | ID позиції у Syrve | Приховане поле `Syrve Item ID` рядка сабформи |
| `items[].productId` | `string (UUID)` | ID продукту у Syrve | Поле рядка сабформи |
| `items[].course` | `integer` | Номер курсу | Поле "Курс" рядка сабформи (конвертований у число, див. нижче) |
| `items[].guestNumber` | `integer` | Номер гостя | Поле "Гість, No" рядка сабформи |

> Поле `city` повинно точно збігатися зі значеннями у нашому списку терміналів: `Khmelnytskyi`, `Lutsk`, `Uzhhorod`. Якщо у вас інша назва -- дайте знати, звіримо.

### Значення поля "Курс"

Якщо у CRM Курс зберігається як текст (picklist), перед надсиланням конвертуйте у число:

| Значення у CRM | Число для API |
|----------------|---------------|
| Без курсу | `0` |
| Перший | `1` |
| Другий | `2` |
| Третій | `3` |
| Четвертий | `4` |

### Поле "Гість"

Передається як ціле число: `1`, `2`, `3`... Якщо позиція не прив'язана до конкретного гостя -- передайте `0`.

### 4.3 Що відповідає наш сервер

**HTTP 200 -- зміни застосовані на касі:**

```json
{
  "status": "success",
  "message": "Order items updated via Front API plugin",
  "orderId": "578f4315-b779-4937-8300-66cbb724b783",
  "itemsCount": 2,
  "terminal": "Khmelnytskyi",
  "timestamp": "2026-03-04 10:00:00"
}
```

**HTTP 400/404 -- каса не знайдена (перевірте значення `city`):**

```json
{
  "status": "error",
  "message": "No terminal found for given City or TerminalName."
}
```

**HTTP 503 -- каса вимкнена або не в мережі:**

```json
{
  "status": "error",
  "message": "Plugin at Khmelnytskyi (10.0.102.30) is unreachable: Connection refused"
}
```

**HTTP 422 -- замовлення вже закрите або оплачене:**

```json
{
  "status": "error",
  "message": "Order status Bill does not allow item updates"
}
```

## 5. Обмеження, які варто знати

| Обмеження | Деталі |
|-----------|--------|
| Тільки активні замовлення | Якщо замовлення у Syrve закрите або оплачене -- наш сервер поверне помилку |
| Каса має бути увімкнена | Якщо каса вимкнена або поза мережею -- команда не дійде |
| Затримка 2-5 секунд | Між натисканням кнопки у CRM і відображенням змін на екрані офіціанта |
| Walk-in замовлення | Якщо замовлення без резервації і в CRM немає відповідного Deal -- синхронізація неможлива |

## 6. Ваша частина реалізації

### 6.1 Визначити API-назви полів

Перед написанням коду зафіксуйте точні API-назви полів у вашій CRM:

| Поле | Де знаходиться | Для чого |
|------|---------------|----------|
| Syrve Order ID | Картка Deal | Передається як `orderId` |
| Місто / Заклад | Картка Deal | Передається як `city` |
| Syrve Item ID | Рядок сабформи (приховане) | Передається як `items[].itemId` |
| Курс | Рядок сабформи | Передається як `items[].course` |
| Гість, No | Рядок сабформи | Передається як `items[].guestNumber` |
| Product ID | Рядок сабформи | Передається як `items[].productId` |

> API-назви можна знайти у: **CRM → Settings → Modules → Deals / Subforms → Fields**.

### 6.2 Написати Deluge-функцію

Функція збирає дані з Deal і сабформи та надсилає запит на наш сервер. Нижче -- структура з коментарями, замініть назви полів на реальні:

```deluge
// 1. Отримати Deal
deal = zoho.crm.getRecordById("Deals", dealId);
orderId = deal.get("Syrve_Order_ID");   // замінити на реальну API-назву
city    = deal.get("City");              // замінити на реальну API-назву

// 2. Зібрати рядки сабформи
subformRows = deal.get("Order_Items");   // замінити на реальну API-назву сабформи
itemsList = list();

for each row in subformRows
{
    courseRaw = row.get("Course");       // picklist або число
    courseNum = courseMapping.get(courseRaw, 0);  // конвертація тексту → число

    item = map();
    item.put("itemId",      row.get("Syrve_Item_ID"));
    item.put("productId",   row.get("Product_ID"));
    item.put("course",      courseNum);
    item.put("guestNumber", row.get("Guest_Number").toLong());
    itemsList.add(item);
}

// 3. Зібрати тіло запиту
payload = map();
payload.put("orderId", orderId);
payload.put("city",    city);
payload.put("items",   itemsList);

// 4. Надіслати запит на наш сервер
headers = map();
headers.put("X-API-Key", "2f72839a224f4544963c9eaa7abcc2f6");
headers.put("Content-Type", "application/json");

response = invokeurl
[
    url: "https://middleware.krainamriy.fun/api/update-order-items"
    type: POST
    parameters: payload.toString()
    headers: headers
];

// 5. Обробити відповідь
status = response.get("status");
if (status == "success")
{
    // показати менеджеру підтвердження
}
else
{
    // показати текст помилки: response.get("message")
}
```

### 6.3 Налаштувати кнопку у Deal

1. **CRM → Settings → Modules → Deals → Links and Buttons** → створити кнопку "Синхронізувати з Syrve"
2. Тип дії: **Invoke a Function**
3. Прив'язати до Deluge-функції з кроку 6.2

### 6.4 Показувати результат менеджеру

Після натискання кнопки бажано виводити зрозуміле повідомлення:

| Відповідь сервера | Що показати менеджеру |
|-------------------|-----------------------|
| `status: success` | "Зміни передані на касу" |
| message містить "unreachable" | "Каса недоступна. Перевірте чи увімкнена." |
| message містить "Bill" | "Замовлення вже оплачене, зміни неможливі." |
| message містить "No terminal found" | "Неправильне місто. Зверніться до адміністратора." |
